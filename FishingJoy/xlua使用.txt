
一、githus上xlua地址：https://github.com/Tencent/xLua

二、xlua导入到工程方法：
1-本地拉取xlua工程/下载xlua压缩包；
2-将xlua文件夹里，Asset文件夹里文件：Plugins，XLua，Plugins.mate，XLua.mate，
复制到工程的Asset文件夹里；
本项目，即把该4个文件复制到：FishingJoyAsset文件夹里，和MyRessources、Scene等文件位于同一层级

三、开启使用xlua的宏
Unity工程里，File-Build Setting->Player Settings->Player->Other Settings->Scripting Define Symbols，
列表新增，输入HOTFIX_ENABLE，按Apply/Enter键确认保存；
这里出现了个对话框，选择了：确定是这些文件，其他文件可能后续再发现；
编译完成后，可以看到Unity菜单栏：Xlua，下拉框多了Hotfix Inject In Editor选项；

四、常见报错处理
1、xlua.access, no field __Hotfix0_Update stack traceback：...
忘记执行：
（1）菜单栏：Xlua-Generate Code；
项目中任何地方有更改，C#有修改，都需要执行该步骤重新生成代码；
（2）菜单栏：Xlua-Hotfix Inject in Editor;
代码生成后，需要把xlua代码注入到编辑器中；
注：打包后不需要执行，但开发补丁过程中需要执行

2、点击Hotfix Inject in Editor，代码注入时，报please install the Tools
将xlua文件夹里，Asset同级目录下的Tools复制到项目工程里；
注：保持文件夹层级一致，
Tools在Xlua里是放到Asset的同级文件夹下的，
所以项目里，也把Tools复制到项目的Asset所在的同级文件夹里；

3、代码注入时，可能出现：
The runtime version supported by this application is unavaliable.
using default runtiome: v4.0.30319
这种警告可以选择忽视，重新点击Hotfix Inject in Editor，代码注入；
如果还注入失败，报这个警告，将三个dll配置文件导入到项目中；（不报警告也可以导）
（1）配置文件所在位置：
Unity安装目录下->Editor->Data->Managed：
（2）目标配置dll：
Unity.Cecil.dll，Unity.Cecil.Mdb.dll，Unity.Cecil.Pdb.dll，
（3）复制到的目标位置：
项目文件夹->Assets->Xlua->Src->Editor文件夹里
注：项目所在目录路径不要有中文，不然可能会因路径名不识别导致异常报错；
